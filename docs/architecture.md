# PhosOpt Architecture

High-level structure of the PhosOpt codebase for cortical visual prosthesis stimulation optimization. Use this document to draw component diagrams and data-flow diagrams.

---

## 1. Overview

**Goal:** Given a **target phosphene map** (desired percept), find **implant placement parameters** and **electrode activations** so that the **forward simulator** produces a phosphene map as close as possible to the target.

**Two main paths:**

- **Learning path (PhosOpt):** Train an **inverse model** (encoder + heads) so that `target → inverse model → (params, electrodes) → simulator → reconstructed map`; optimize with a **differentiable simulator** and **loss**.
- **Optimization path (baselines):** For each target, run an optimizer (e.g. Bayesian, heuristic) that repeatedly calls the **simulator** and uses **evaluation metrics** to improve params and electrode mask.

---

## 2. Data: Target Maps

| Concept | Description |
|--------|-------------|
| **Target map** | 2D array (e.g. 256×256 or 1000×1000), float32, typically in [0,1]. Desired phosphene density pattern in visual field (right hemifield, -90° to 90°). |
| **Source** | Generated by `code/targets/generator.py` (single/multi blob, arc, MNIST letters) or loaded from `data/targets/*.npy`. |
| **Naming** | e.g. `single_blob_00`, `arc_02`, `mnist_letter_T`. |

Target maps are the **input** to both the inverse model (as images) and to baseline optimizers (as objective to match).

---

## 3. Inverse Model (PhosOpt)

Single neural model that predicts **one shared implant** and **per-image electrode activations**.

```
Target map [B, 1, H, W]
       │
       ▼
┌──────────────────┐
│     Encoder      │  (e.g. CNN → latent_dim)
└────────┬─────────┘
         │ latent [B, latent_dim]
         ├──────────────────────────────┐
         ▼                              ▼
┌─────────────────────┐    ┌─────────────────────────────────────┐
│  Shared params      │    │  Electrode head                     │
│  (single vector)    │    │  latent → 1000 logits (per image)  │
│  [4]                │    │  → sigmoid → electrode activations │
└─────────┬───────────┘    └─────────────────┬─────────────────┘
          │                                   │
          └───────────────┬───────────────────┘
                          ▼
              (params [B,4], electrode_activations [B,1000])
```

### 3.1 Shared implant parameters (4)

| Index | Name | Meaning | Bounds (typical) |
|-------|------|---------|-------------------|
| 0 | **alpha** | Implant angle (deg) | [-90, 90] |
| 1 | **beta** | Implant angle (deg) | [-15, 110] |
| 2 | **offset** | Offset from base (mm) | [0, 40] |
| 3 | **shank_length** | Shank length (mm) | [10, 40] |

- Stored as one **global** `nn.Parameter` (same for all images): one physical implant.
- Bounded via sigmoid and passed to the simulator.

### 3.2 Electrode activations (1000)

- **Electrode head** outputs 1000 **logits** per image.
- **sigmoid(logits)** → activations in [0, 1] (per-electrode weight).
- **1000** = fixed count from the implant grid (basecode: 10 contacts per shank × 100 shanks → 1000 contact points).

### 3.3 Code locations

- `code/models/inverse_model.py`: `InverseModel` (Encoder + shared params + ElectrodeHead).
- `code/models/encoder.py`: CNN encoder (e.g. 256×256 → latent).
- `code/models/parameter_head.py`: `ParameterBounds`, `ElectrodeHead` (1000 dim), `ContinuousHead` (4 dim, used elsewhere).

## 4. Forward Simulator (Phosphene map from params + electrodes)

The simulator maps **(4 params, 1000 electrode activations)** → **phosphene map (2D)**.

```
(params [4], electrode_activation [1000])
       │
       ▼
┌──────────────────────────────────────────────────────────────────┐
│  Retinotopy / anatomy (subject-specific, preloaded)               │
│  - polar_map, ecc_map, sigma_map (PRF properties)                  │
│  - V1 coords, gray matter mask, start_location                    │
└──────────────────────────────────────────────────────────────────┘
       │
       ▼
┌──────────────────────────────────────────────────────────────────┐
│  Implant placement                                                │
│  - create_grid(start_location, shank_length, n_contactpoints)    │
│  - implant_grid(gm_mask, grid, start_location, (alpha, beta), offset) │
│  → contacts_xyz_moved [3, 1000]                                   │
└──────────────────────────────────────────────────────────────────┘
       │
       ▼
┌──────────────────────────────────────────────────────────────────┐
│  Contact → V1 mapping                                             │
│  - build_weighted_phosphenes(contacts_xyz, good_coords,            │
│      polar_map, ecc_map, sigma_map, contact_weights)               │
│  → phosphene list (polar, ecc, sigma) + weights                    │
└──────────────────────────────────────────────────────────────────┘
       │
       ▼
┌──────────────────────────────────────────────────────────────────┐
│  Cortical magnification + PRF size                                │
│  - get_cortical_magnification(ecc), cortical_spread(AMP)          │
│  - adjust phosphene size (sigma)                                   │
└──────────────────────────────────────────────────────────────────┘
       │
       ▼
┌──────────────────────────────────────────────────────────────────┐
│  Render to 2D map                                                 │
│  - prf_to_phos_weighted(phosphene_map, phosphenes, weights)        │
│  - Gaussian splatting in visual field (view_angle=90°)             │
│  - normalize_phosphene_map → [0,1]                                │
└──────────────────────────────────────────────────────────────────┘
       │
       ▼
Phosphene map [H, W]  (e.g. 1000×1000 or 256×256)
```

### 4.1 Two simulator backends

| Backend | Module | Differentiable? | Use |
|---------|--------|------------------|-----|
| **Numpy** | `physics_forward.make_phosphene_map` + `NumpySimulatorAdapter` | No | Inference, baselines (Bayesian, heuristic). |
| **Torch** | `physics_forward_torch.DifferentiableSimulator` | Yes | Training inverse model (gradients w.r.t. params and electrode logits). |

- **Numpy:** Uses basecode `ninimplant.get_xyz`, `electphos.create_grid`, `implant_grid`, `get_phosphenes`, `prf_to_phos`-style rendering.
- **Torch:** Approximates the same pipeline with soft matching and differentiable Gaussian rendering so that gradients flow through params and electrode activations.

### 4.2 Code locations

- `code/simulator/physics_forward.py`: `make_phosphene_map`, `load_retinotopy`, `build_weighted_phosphenes`, `prf_to_phos_weighted`.
- `code/simulator/physics_forward_torch.py`: `DifferentiableSimulator`.
- `code/simulator/simulator_wrapper.py`: `SimulatorWrapper` (diff), `NumpySimulatorAdapter` (numpy).

### 4.3 Inputs the simulator needs

- **Subject data dir** (e.g. `data/fmri/100610`): inferred angle/eccen/sigma/varea (retinotopy).
- **Hemisphere**: LH or RH.
- **4 params** + **1000 electrode activations** (float in [0,1]).

## 5. Loss (Training the inverse model)

Used only when training with the **differentiable simulator**.

| Term | Role |
|------|------|
| **Reconstruction (MSE)** | `(recon - target)^2` |
| **Dice loss** | Soft Dice (1 - Dice) between reconstructed and target map |
| **Sparsity** | Mean electrode activation (encourage few active electrodes) |
| **Param prior** | Keep shared params near their mean (regularization) |
| **Invalid region** | Penalize electrode activation on invalid cortex (if mask provided) |

**Total loss** = weighted sum of the above (with optional warmup for sparsity/prior/invalid).

- **Code:** `code/loss/losses.py` (`build_losses`, `soft_dice_loss`, etc.).

---

## 6. Evaluation Metrics (Benchmarks & Reporting)

Used to **score** a (params, electrode_mask) solution against a target map (and optionally anatomy).

| Metric | Meaning |
|--------|--------|
| **DC** | Dice coefficient (overlap of binarized pred vs target) |
| **Y (Yield)** | Fraction of electrode contacts that land in valid cortex |
| **HD** | Hellinger distance between (normalized) density maps |
| **Composite score S** | S = DC + 0.1×Y − HD |
| **Loss (vimplant)** | 2 − S |

Efficiency metrics: **simulator calls**, **wall-clock time**, **active electrode count**.

- **Code:** `code/metrics/eval_metrics.py`, `code/baselines/vimplant_cost.py` (for yield and vimplant cost).

## 7. End-to-end Flows

### 7.1 Training (PhosOpt)

1. **Target maps** (from DataLoader, e.g. from `data/targets/*.npy` or npz).
2. **Inverse model** forward: target → (shared_params, electrode_logits).
3. **Simulator** forward: (params, sigmoid(electrode_logits)) → reconstructed map.
4. **Loss** = f(reconstructed, target, params, electrode_logits, …).
5. Backprop through simulator + inverse model; update encoder, electrode head, and shared params.

### 7.2 Per-target optimization (baselines)

1. **Target map** (single map).
2. **Optimizer** (Bayesian, all-on, random subset, heuristic, or PhosOpt per-target) proposes (params, electrode_mask).
3. **Simulator** (numpy) produces phosphene map.
4. **Evaluation metrics** (DC, Y, HD, S) compare map to target; optionally vimplant cost with yield/penalty.
5. Repeat until budget (simulator calls / time) exhausted.

### 7.3 Generalized benchmark (shared params)

1. **Multiple target maps** (train/val/test split).
2. One **shared (4) params** and **per-target electrode activations** (e.g. from a trained inverse model or from optimization).
3. For each target, run **simulator** with shared params + that target’s electrodes; compute metrics.
4. Report aggregate (e.g. mean S) over targets.

### 7.4 Adaptation benchmark

1. **Unseen target maps** (test set).
2. Start from a pre-trained inverse model (or fixed shared params); **adapt** (e.g. fine-tune electrodes or run optimizer) for each new target.
3. Compare methods by reconstruction quality and efficiency (simulator calls, time).

## 8. Config & Entry Points

| Entry | Role |
|-------|------|
| **experiment.py** | Reads YAML config; builds simulator, loads targets; runs per_target / generalized / adaptation benchmark. |
| **train.py** | Trains inverse model (encoder + shared params + electrode head) with dataloader, differentiable simulator, and loss. |
| **targets/generator.py** | Generates target maps (blob, arc, letters) and saves as `.npy`. |

Config (YAML) typically specifies: `simulator.backend` (numpy / diff), `simulator.retinotopy_dir`, `simulator.hemisphere`, `simulator.map_size`, `targets` (categories, splits), and benchmark-specific options.

## 9. Diagram-oriented summary (boxes and arrows)

- **Input:** Target map(s) `[B, 1, H, W]`.
- **Inverse model:** Target → **Encoder** → latent → **Electrode head** → 1000 logits; **shared params** 4 dims.
- **Activations:** sigmoid(electrode_logits) → 1000-dim vector in [0,1].
- **Simulator:** (4 params, 1000 activations) + retinotopy/anatomy → **Phosphene map** [H,W].
- **Loss:** recon + Dice + sparsity + param prior + invalid_region (training only).
- **Evaluation:** DC, Yield, HD, composite S (and efficiency metrics).

Relationships:

- Target maps **drive** the inverse model and **define the objective** for the simulator output.
- The **same 4 params** and **same 1000-electrode grid** are used in both numpy and torch simulators; only the implementation (diff vs non-diff) differs.
- Loss connects **reconstructed map** to **target map** and regularizes **params** and **electrode activations**.
- Metrics compare **simulator output** to **target** (and optionally use **contacts_xyz** and **good_coords** for Yield).

This document is intended to be stable for drawing **architecture and data-flow diagrams**; refer to the listed code paths for exact APIs and constants.

**Config detail:** YAML uses `budget.max_wall_clock_min`; `experiment.run_per_target` converts it to seconds when building `BudgetConfig.max_wall_clock_sec`.

**Target filtering:** In `_load_targets`, when `targets.indices` is a list (e.g. `[0, 1]`), only targets whose suffix parses as an integer are kept (e.g. `single_blob_00`). Letter-named targets like `mnist_letter_T` are included only when `indices: "all"` (or when categories filter leaves them and no integer filter is applied). If you use letter names, set `indices: "all"` or omit indices.
